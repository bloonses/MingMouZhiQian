# 明眸智签 - 使用说明

## 🚀 快速启动

### 账号说明
- **初始管理员**：用户名 `admin`，密码 `admin123`（首次启动自动创建）
- **注册新账户**：点击登录页"创建新账户"，注册后默认为教师角色
- 超级管理员和普通教师拥有不同的管理权限

### 第一次使用
1. 双击 **环境配置.bat** 安装依赖（含 InsightFace 人脸识别引擎）
2. 双击 **一键启动.bat** 启动系统

### 后续使用
直接双击 **一键启动.bat** 即可

### 两种识别模式
系统支持**前端/后端双模式**，在签到页面点击「模式: 前端」按钮切换：

| 模式 | 技术栈 | 特征维度 | 活体检测 | 说明 |
|------|------|----------|----------|------|
| 前端模式 | face-api.js（浏览器） | 128维 | 点头验证 | 无需额外配置，开箱即用 |
| 后端模式 | InsightFace（Python） | 512维 | 鼻尖移动验证 | 速度更快（~50ms/帧），精度更高（LFW 99.8%），支持多人检测 |

## 📋 功能模块

### 1. 班级管理
- 添加、编辑、删除班级
- **导入/导出Excel**：支持批量导入导出班级数据
- 点击左侧菜单"班级管理"

### 2. 学生管理
- 添加、编辑、删除学生
- **采集人脸**：为每个学生采集人脸数据（同时采集 128 维和 512 维双特征）
- **导入/导出Excel**：支持批量导入导出学生数据
- 按班级筛选学生
- 点击左侧菜单"学生管理"

### 3. 人脸识别签到（单人模式）
- 选择班级/课程自动启动摄像头
- **活体检测**：
  - 前端模式：需要点头才能完成签到
  - 后端模式：检测到人脸后自动记录鼻尖移动，移动距离超过阈值即通过活体
- **双模式切换**：点击「模式: 前端/后端」在 face-api.js 和 InsightFace 之间切换
- 支持手动签到（点击学生旁边的📝按钮）
- 签到成功时显示"姓名+已签到"提示弹窗 + 绿光涟漪动画
- 清除当日签到数据
- 点击左侧菜单"人脸识别签到"

### 4. 批量签到（多人模式）
- 选择课程自动启动摄像头
- 同时检测多名学生人脸
- 识别人脸后自动完成签到
- 显示实时签到统计（应到/已到/未到）
- 支持手动签到补充
- 清除当日课程签到数据

### 5. 课程管理
- 添加、编辑、删除课程
- 关联多个班级到课程
- 点击左侧菜单"课程管理"

### 6. 二维码签到（备用方案）
- **教师端**：生成动态二维码（每10秒自动刷新，防代签）
- **学生端**：用手机扫码 → 输入学号 → 提交签到请求
- **教师确认**：查看待确认列表 → 点击确认/拒绝 → 完成签到
- 支持人脸识别和二维码签到互斥（同一学生同一天只能签到一次）
- 点击左侧菜单"二维码签到"

### 7. 签到记录
- 查看所有签到历史
- 按班级和日期筛选
- 点击左侧菜单"签到记录"

### 8. 数据统计
- 查看多维度签到统计图表（折线图、柱状图、饼图）
- 按班级查看统计数据
- **导出统计Excel**：包含总体统计、班级统计、详细记录
- 点击左侧菜单"数据统计"

## 🎯 活体检测使用说明

### 什么是活体检测？
系统采用**两种活体检测方式**防止使用照片或视频进行作弊签到：

| 模式 | 活体检测原理 |
|------|-------------|
| 前端模式 | 点头检测：跟踪鼻尖 Y 轴移动，检测是否有点头动作 |
| 后端模式 | 鼻尖移动检测：跟踪鼻尖在连续帧中的移动距离，真人会自然微动头部 |

### 前端模式：点头检测
1. 选择班级后，摄像头会自动启动
2. 确保面部正对摄像头
3. **自然地点头一次**（上下移动头部）
4. 检测通过后，系统会自动识别并完成签到

### 后端模式：鼻尖移动检测
1. 选择班级后，摄像头会自动启动
2. 确保面部正对摄像头
3. **轻微移动头部**（左右或上下，不需要点头）
4. 系统记录鼻尖移动，当移动距离超过阈值后活体通过
5. 活体通过后立即开始人脸匹配

## ⚠️ 注意事项

### 摄像头权限
- 首次使用时，浏览器会弹出摄像头权限请求
- **必须点击"允许"**才能使用人脸识别功能
- 如果没有弹出，请检查浏览器地址栏左侧的🔒图标

### 人脸采集要求
- 光线充足，避免逆光
- 正对摄像头，表情自然
- 确保能看到绿色的人脸检测框
- 采集成功后会提示「人脸采集成功（128维+512维双特征）」

### 签到环境要求
- 光线充足，避免强逆光或暗光
- 人脸与摄像头保持合适距离（0.5-2米）
- 面部正对摄像头
- 签到时自然移动头部（后端模式）或点头（前端模式）

### 导入注意事项
- 仅支持 `.xlsx` 格式的Excel文件
- 导入前请确保Excel格式正确
- 已存在的班级/学生不会重复导入

## 常见问题

**Q: 摄像头无法打开？**
A: 
- 检查浏览器是否授予了摄像头权限
- 确认没有其他程序占用摄像头
- 尝试在浏览器设置中允许摄像头访问
- 必须使用 `localhost` 或 `127.0.0.1` 访问

**Q: 人脸识别不准确？**
A: 
- 确保已为所有学生采集了人脸数据
- 采集时尽量在光线良好的环境下进行
- 识别时确保面部正对摄像头
- 可参考 [关键参数文档.md](关键参数文档.md) 调整识别阈值
- 建议采集和识别使用同一模式（都用后端或都用前端）

**Q: 点头检测没反应（前端模式）？**
A:
- 确保人脸在画面中且距离合适
- 自然地点头，动作不要太快或太慢
- 点击"🔧 调试"按钮查看详细检测信息
- 可参考 [关键参数文档.md](关键参数文档.md) 调整检测参数

**Q: 后端模式提示「请轻微移动头部」？**
A:
- 这是活体检测提示，表明正在跟踪鼻尖移动
- 请自然轻微移动头部（左右或上下）
- 收集 6 帧鼻尖数据后会自动判断活体是否通过
- 活体通过后会立即开始人脸匹配

**Q: 如何清除当日的签到数据？**
A: 在签到页面，点击"🗑️ 清除签到"按钮

**Q: 导入Excel失败？**
A: 
- 检查文件格式是否为 `.xlsx`
- 检查Excel表头是否正确
- 确保必填字段不为空

**Q: 如何调整识别准确率？**
A:
- 详细参数说明请参考 [关键参数文档.md](关键参数文档.md)
- 后端模式使用余弦相似度，阈值为 0.45
- 前端模式使用欧氏距离，阈值为 0.6

**Q: 二维码签到如何防止代签？**
A:
- 二维码每10秒自动刷新，旧码立即失效
- 学生扫码后不会立即签到成功，需教师端手动确认
- 同一学生同一天只能签到一次，人脸识别签到后二维码签到无效（反之亦然）

**Q: 手机扫码后显示无法打开？**
A:
- 确保手机和电脑在同一局域网内
- 二维码链接会自动使用当前服务器的 IP 地址
- 如果仍无法访问，请检查防火墙设置

**Q: 如何导出签到数据？**
A:
- 在班级管理/学生管理页面点击"📤 导出Excel"
- 在数据统计页面点击"📊 导出统计"获取完整统计分析

## 🔧 技术支持

- 操作系统：Windows 10/11
- 浏览器推荐：Chrome、Edge、Firefox
- Python版本：3.8+
- 核心依赖：Flask、SQLAlchemy、openpyxl、numpy、qrcode
- 人脸识别：face-api.js（前端） + InsightFace / ONNX Runtime（后端）
- 后端模型：InsightFace buffalo_l（RetinaFace 检测 + ArcFace 特征提取）
- 特征维度：128 维（前端） / 512 维（后端）
- 匹配算法：余弦相似度（后端模式） / 欧氏距离（前端模式）
