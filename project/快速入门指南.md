# 明眸智签 - 快速入门指南

## 🚀 5分钟快速开始

### 步骤1：运行项目
双击 `一键启动.bat`，访问 http://127.0.0.1:5000

### 步骤2：创建班级
点击"班级管理" → "添加班级"，输入班级信息
**或者**：点击"📥 导入Excel"批量导入班级

### 步骤3：添加学生
点击"学生管理" → "添加学生"，选择班级，输入学生信息
**或者**：选择班级后点击"📥 导入Excel"批量导入学生

### 步骤4：采集人脸
点击学生列表中的"采集人脸"按钮，按提示操作
- **确保采集成功（128维+512维双特征）** 才可以使用后端模式识别
- 如果仅采集到128维，仅能使用前端模式

### 步骤5：开始签到（支持双模式）
点击"人脸识别签到"，选择班级和课程
- 系统自动启动摄像头
- **前端模式**：点头完成活体检测
- **后端模式**：轻微移动头部完成活体检测
- 识别成功后会显示"姓名+已签到"提示弹窗 + 绿光涟漪动画
- **点击「模式: 前端」可切换为后端 InsightFace 识别**（更快更准）

### 步骤6：二维码签到（备用）
点击"二维码签到"，选择课程
- 教师端展示动态二维码（每10秒自动刷新）
- 学生用手机扫码 → 输入学号 → 提交请求
- 教师在待确认列表中点击确认完成签到
- 同一学生人脸签到后二维码签到无效（防代签）

### 步骤7：批量签到
点击"批量签到"，选择课程
- 系统自动启动摄像头
- 同时检测多名学生
- 识别人脸后自动完成签到
- 支持手动签到补充

### 步骤8：查看统计
点击"数据统计"，选择班级查看多维度分析图表

### 步骤9：导出数据
- 班级管理：点击"📤 导出Excel"下载班级信息
- 学生管理：选择班级后点击"📤 导出Excel"下载学生信息
- 数据统计：点击"📊 导出统计"下载完整签到分析

## 🐛 常见开发问题

### 问题1：修改代码后页面没变化？
**可能原因：**
- 浏览器缓存了旧页面
- Flask还没重新加载

**解决方法：**
1. 强制刷新：Ctrl + Shift + R（Windows）
2. 或者清除浏览器缓存
3. Flask Debug模式开启时会自动重新加载

### 问题2：数据库报错？
**可能原因：**
- 数据库结构与代码不匹配（比如缺少 `face_descriptor_512` 列）

**解决方法：**
1. 删除 `instance/attendance.db`
2. 重新运行项目，会自动创建新数据库
3. **注意**：这样会丢失所有数据！

### 问题3：face-api.js加载失败？
**检查：**
1. 确保 `static/models/` 目录下有模型文件
2. 查看浏览器控制台的Network标签
3. 确认模型文件路径正确

### 问题4：摄像头无法访问？
**注意：**
- 必须用 `localhost` 或 `127.0.0.1` 访问
- 不能直接打开HTML文件
- 浏览器必须允许摄像头权限

### 问题5：后端模式无法使用？
**检查：**
1. 确认 insightface 已安装：`pip show insightface`
2. 确认模型已下载到 `.insightface/models/buffalo_l/`
3. 查看终端控制台输出，确认显示「InsightFace 模型初始化成功」
4. 如果 SSL 下载失败，可手动下载模型并解压到项目目录
5. **确保采集了512维特征**（采集时提示「双特征」）

### 问题6：后端模式识别不出来？
**检查：**
1. 查看终端日志中的 `cos_sim` 值
2. 如果值低于0.45，考虑降低阈值
3. 确认采集和识别使用同一模式
4. 确认活体检测通过了（终端显示 `liveness=True`）

### 问题7：下拉选择栏无法使用？
**已修复！**
- 更新了 [templates/base.html](templates/base.html) 中的下拉菜单样式
- 增加了z-index层级
- 优化了深色主题下的显示效果

### 问题8：切换班级/模式后摄像头画面不动？
**已修复！**
- 现在切换时会重新初始化 canvas 和 ctx
- 确保视频流始终正常显示

## 💡 开发建议

### 1. 先学会使用，再开始开发
先完整使用一遍系统，理解每个功能做什么的

### 2. 小步快跑
不要一次性改很多代码，改一点测试一点

### 3. 多打印日志

每次完成一个功能，做一次Git提交，方便回退
