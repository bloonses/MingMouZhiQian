**余弦相似度公式：**
```
similarity = dot(a, b) / (norm(a) * norm(b))
```
- 范围：-1 ~ 1
- 值越大，越相似
- 阈值 0.45 是经验值

**调优建议：**
- **漏识别（识别不出来）**：降低阈值到 0.35-0.40
- **误识别（识别错人）**：提高阈值到 0.55-0.60

## 🔍 快速调试指南

### 问题诊断流程

#### 1. 识别不准确（漏识别）

**可能原因：**
- 匹配阈值过高
- 检测置信度过高
- 摄像头分辨率不足
- 采集和识别使用了不同模式（前端采集 + 后端识别）

**调优步骤：**
1. 确保采集和识别使用同一模式（都用后端或都用前端）
2. 降低匹配阈值：后端（余弦 >0.35），前端（距离 <0.5）
3. 调整 scoreThreshold 让检测更敏感
4. 开启调试面板查看匹配距离/相似度

#### 2. 误识别频繁

**可能原因：**
- 匹配阈值过低
- 检测置信度过低
- 相似人脸过多

**调优步骤：**
1. 提高匹配阈值：后端（余弦 >0.55），前端（距离 <0.65）
2. 提高 scoreThreshold 减少误检
3. 检查是否有多张相似人脸

#### 3. 后端模式提示「请轻微移动头部」

**可能原因：**
- 活体检测正在收集数据
- 鼻尖移动阈值过高
- 摄像头不动（用照片）

**调优步骤：**
1. 请自然轻微移动头部
2. 降低活体移动阈值到 5-8 像素
3. 检查终端日志中的 `nose_frames` 计数

#### 4. 点头检测不灵敏（前端模式）

**可能原因：**
- 点头阈值过高
- 历史窗口过大
- 人脸距离摄像头过远

**调优步骤：**
1. 降低移动阈值到 5-8 像素
2. 减小最小帧数到 10 帧

#### 5. 点头误检测（前端模式）

**可能原因：**
- 点头阈值过低
- 头部微动被误判

**调优步骤：**
1. 提高移动阈值到 15-20 像素
2. 增加最小帧数到 20 帧

#### 6. 切换班级/模式后摄像头画面不动

**已修复！**
- 现在切换时会重新初始化 canvas 和 ctx
- 确保视频流始终正常显示

### 开启调试面板

1. 打开签到页面
2. 点击右上角的 **"🔧 调试"** 按钮
3. 查看实时调试信息输出

### 后端调试日志

查看 Python 终端输出：
- `[Recognize] student_id=X cos_sim=0.XX distance=0.XX`：匹配信息
- `[Recognize] 活体检测: nose_frames=X, liveness=True/False`：活体状态
- `[SaveFace] 人脸采集成功（128维+512维双特征）`：采集成功
- `[SaveFace] 人脸采集成功（仅128维，后端识别不可用）`：仅前端模式可用

### 浏览器开发者工具

1. 按 `F12` 打开开发者工具
2. 切换到 **Console** 标签
3. 查看详细的控制台日志输出

### 查看网络请求

1. 打开开发者工具（F12）
2. 切换到 **Network** 标签
3. 筛选 Fetch/XHR 请求
4. 查看 `/api/recognize_face` 或 `/api/recognize_frame` 请求和响应
